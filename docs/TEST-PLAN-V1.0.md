{
  "results": [

## 四、数据库权限设置（关键步骤）

进入「云开发控制台 → 数据库」，对每个集合点击「权限设置」，选择 **"自定义权限"**，粘贴以下配置：

```json
{
  "read": true,
  "write": "doc._openid == auth.openid"
}
```

### 4.1 权限设置清单

| 集合名 | 权限配置 | 状态 |
|--------|----------|------|
| `patients` | read: true, write: doc._openid == auth.openid | ⬜ |
| `timeline` | read: true, write: doc._openid == auth.openid | ⬜ |
| `indicators` | read: true, write: doc._openid == auth.openid | ⬜ |
| `reminders` | read: true, write: doc._openid == auth.openid | ⬜ |
| `familyMembers` | read: true, write: doc._openid == auth.openid | ⬜ |
| `operations` | read: true, write: doc._openid == auth.openid | ⬜ |

### 4.2 权限说明

- **read: true**：任何人可读取，确保家庭成员能查看数据
- **write 限制**：仅创建者可通过小程序端直接修改，防止越权
- **云函数写入**：addRecord、getHomeData 等云函数拥有管理员权限，不受 write 限制，确保家庭成员可通过云函数正常添加记录
