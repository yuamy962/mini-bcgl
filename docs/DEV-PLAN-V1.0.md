# 病程管家（MVP版）开发计划

> 版本：V1.0  
> 基于需求文档：REQ-V1.0.md + 流程以及设计.md  
> 核心原则：**先完成记录→查看→提醒的完整闭环，再逐步增强**

---

## 一、产品核心闭环

```
用户打开小程序
    │
    ▼
┌─────────────┐
│   首页      │  ← 病程控制中心：3秒知道病情
│  Dashboard  │     当前状态 / 即将到期提醒 / 最近摘要 / 快速入口
└──────┬──────┘
       │
   ┌───┴───┐
   │       │
   ▼       ▼
快速记录  查看/管理
   │       │
   ▼       ▼
时间轴 ←──┘
   │
   ├──► 自动生成提醒
   │
   └──► 更新趋势分析
```

**闭环标准**：用户能在 10 秒内完成一次记录，并在首页立刻看到病情变化。

## 三、技术架构

| 层级 | 技术方案 |
|------|----------|

## 四、数据库设计（云开发数据库）

| 集合名 | 用途 | 核心字段 |
|--------|------|----------|
| `patients` | 患者档案 | `_openid`, `name`, `age`, `gender`, `cancerType`, `diagnosisDate`, `treatmentStatus`, `hospital`, `doctor`, `createTime` |
| `timeline` | 病程时间轴 | `_openid`, `date`, `type`, `title`, `value`, `unit`, `hospital`, `drug`, `doctorAdvice`, `images`, `remark`, `createTime` |
| `indicators` | 指标记录 | `_openid`, `indicatorType` (TPSA/FPSA), `value`, `date`, `hospital`, `createTime` |
| `reminders` | 提醒配置 | `_openid`, `type`, `title`, `nextDate`, `cycleDays`, `isActive`, `sourceEventId`, `createTime` |
| `familyMembers` | 家庭成员 | `patientId`, `inviterOpenid`, `memberOpenid`, `nickName`, `role`, `permission`, `createTime` |
| `operations` | 操作记录 | `patientId`, `operatorOpenid`, `operatorName`, `action`, `content`, `createTime` |

**权限策略**：
- `patients`、`timeline`、`indicators`、`reminders`：创建者可读写，关联家庭成员可读（根据 `familyMembers.permission`）
- `familyMembers`：创建者可读写，被邀请者可读
- `operations`：创建者可读写，关联家庭成员可读

## 七、云函数清单

| 云函数 | 功能 | 说明 |
|--------|------|------|
| `addRecord` | 统一添加记录 | 接收类型和表单数据，写入 `timeline`，同步写入 `indicators`（如为PSA记录），生成 `reminders`（如为打针记录），写入 `operations` |
| `updateRecord` | 更新/删除记录 | 同步维护关联数据 |
| `getHomeData` | 聚合首页数据 | 一次查询返回：最近PSA、当前状态、即将到期提醒、最近摘要 |
| `getSummary` | 生成完整病程摘要 | 聚合最近指标、治疗、检查、建议、到期事项 |
| `sendReminder` | 触发订阅消息提醒 | 云函数定时触发器，每日检查到期提醒 |
| `generateShareImage` | 生成分享长图 | 基于病程摘要用 Canvas 绘制长图 |
| `inviteFamily` | 生成家庭邀请 | 生成邀请码/二维码 |
| `acceptFamilyInvite` | 接受邀请 | 验证并加入家庭 |

### 阶段二：快速记录 + 时间轴（1.5 天）

**目标**：用户能 10 秒内完成一次记录，并在时间轴看到。

| 序号 | 任务 | 产出 |
|------|------|------|
| 2.1 | 开发 `components/float-add-btn`：首页底部悬浮 "+" | 点击弹出选择面板 |
| 2.2 | 开发 `components/record-type-sheet`：6种类型选择面板 | 底部弹出交互完成 |
| 2.3 | 开发 6 个类型表单组件（PSA/检查/打针/用药/复诊/备注） | 每种表单极简，10秒内完成 |
| 2.4 | 打针记录表单：保存后**自动生成**下次打针提醒 | 提醒系统自动初始化 |
| 2.5 | 开发 `pages/timeline/index`：时间轴列表页 | 按日期倒序，日期分组 |
| 2.6 | 开发 `components/timeline-card` + `timeline-date-divider` | 卡片展示所有字段 |
| 2.7 | 开发 `pages/timeline/detail`：事件详情/编辑/删除 | 可查看完整信息 |
| 2.8 | 图片上传：检查记录支持上传检查报告到云存储 | 图片可预览 |

**表单设计原则**：
- PSA记录：类型选择 → 输入数值 → 选择日期 → 保存（4步）
- 检查记录：检查类型 → 医院（可选）→ 备注/图片 → 保存
- 打针记录：药物选择 → 日期 → **自动计算下次日期** → 保存
- 用药记录：药物名称 → 开始时间 → 下次拿药时间 → 保存
- 复诊记录：医院 → 医生建议 → 后续方案 → 保存
- 备注记录：文本输入 → 保存（2步）

**里程碑**：首页点击"+" → 选类型 → 填表单 → 保存 → 自动跳转到时间轴看到新记录。
